**参照**: `E:\manual-v2.pdf`（システムマニュアル Ver 1.2）  
**目的**: マニュアルで定義された業務フロー・機能を、Supabase をバックエンドとする Ver.2 で実装するための計画です。

## 1. システム概要（マニュアル CH01 対応）

### 1.1 基本方針
- **データベース**: Google Spreadsheet → **Supabase（PostgreSQL）** に移行
- **フロント**: スマートフォン・PC 両対応の Web アプリ（推奨: React / Next.js 等）
- **業務範囲**: 受注管理・製造計画・在庫管理・入出荷管理の一元化

### 1.2 基本業務フロー（維持）
```
STEP1 受注登録 → STEP2 製造計画入力 → STEP3 製造中→完了 → STEP4 出荷実行
```
- 入荷管理は製造開始前と並行実施
- 在庫が安全在庫を下回ったらダッシュボードにアラート表示

### 1.3 ページ構成（実装対象）

| ページ | 役割 | 優先度 |
|--------|------|--------|
| ダッシュボード | 在庫不足アラート・本日の予定・棚卸履歴 | 高 |
| 受注管理 | 新規受注登録・受注履歴・資材シミュレーション | 高 |
| 製造管理 | 計画入力モード / カレンダーモード（予定表・編集） | 高 |
| 在庫管理 | 原材料/資材/製品在庫・棚卸 | 高 |
| 入荷管理 | 入荷予定登録・受入完了・印刷カレンダー | 高 |
| 出荷管理 | 出荷実行・ロット選択・在庫減算 | 高 |
| マスタ・設定 | 製品・品目・BOM・出荷先の編集案内 | 中 |

## 2. データモデル設計（Supabase）

### 2.1 マスタ系
- **製品マスタ**: 製品ID, 製品名, 製造種類, 1kg個数, 1c/s個数
- **品目マスタ**: 品目ID, 品目名, 品目区分（原材料/資材）, 規格量, 単位, 安全在庫数
- **BOM**: 製品ID, 品目ID, 使用率, 基準単位（製造量 kg / 受注数 c/s）
- **出荷先マスタ**: 出荷先ID, 出荷先名

### 2.2 トランザクション系
- **受注**: 受注ID（ORD-YYYYMMDD-001）, 受注日, 出荷先ID, 製品ID, 受注数(c/s), 希望出荷日, ステータス（受注済/製造中/完了/出荷済）
- **製造計画**: 計画ID（PLN-xxxxxxxx）, 受注ID, 製造予定日, 製造量(kg), 製造個数・製造数(c/s), 賞味期限, ステータス（計画/製造中/完了）, Lot番号
- **品目在庫**: 品目ID, 現在庫, 棚卸調整履歴と連動
- **製品在庫（ロット別）**: 製品ID, Lot番号, 製造日, 賞味期限, 数量(c/s, p)
- **入荷予定/実績**: 入荷ID（INC-YYYYMMDD-001）, 品目ID, 入荷予定日, 数量, ステータス（未入荷/入荷済）, 入荷日
- **出荷**: 受注ID, 出荷日, ロット別出荷数, 出荷管理シート相当のレコード

### 2.3 ルール・採番
- 受注ID: `ORD-YYYYMMDD-001` 形式の自動採番
- 製造計画ID: `PLN-xxxxxxxx` 形式
- 入荷ID: `INC-YYYYMMDD-001` 形式
- Lot番号: 製品ID + 製造日から自動（例: 01A26YC）。アソート(MA/FD)は手動採番（例: 26MA01、年度は4月始まり）

## 3. 機能別実装計画

### CH02 受注管理
- [ ] 受注登録フォーム: 受注日・出荷先・製品名→製造種類・受注数・希望出荷日
- [ ] 出荷先はテキスト入力＋候補選択で出荷先ID自動セット
- [ ] 製品名選択で製造種類を絞り込み、製品ID確定
- [ ] 右パネル: 必要資材シミュレーション（BOM展開、MA/FDは子製品内訳表示）
- [ ] 受注一覧・ステータス表示（受注済/製造中/完了/出荷済）

### CH03 製造管理
- [ ] 計画入力モード: 未計画受注を選択し、製造予定日・製造量(kg)を登録。複数日は「＋予定追加」
- [ ] 製造量→製造個数・製造数(c/s)の自動計算、賞味期限は製造日+5年6ヶ月
- [ ] アソート製品は子製品に分解して個別計画登録
- [ ] 予定表・編集モード: 月別カレンダー、ドラッグ＆ドロップで製造日変更
- [ ] ステータスボタン: 計画→製造中→完了。製造中で原材料・資材減算、完了で製品在庫加算＋実績個数入力
- [ ] 印刷用表示（A4横向き）

### CH04 在庫管理
- [ ] カテゴリ: 原材料 / 資材 / 製品在庫
- [ ] 表示: 現在庫・使用予定・有効在庫（現在庫−使用予定）、適正/在庫低下/欠品警告
- [ ] 品名検索でリアルタイム絞り込み
- [ ] 棚卸: 「棚卸開始」で実在庫入力欄表示、保存確定で在庫更新＋棚卸調整履歴記録。URLパラメータ `?inventory=true` で棚卸モード起動

### CH05 入荷管理
- [ ] 入荷予定登録: 品目選択（原材料/資材）、入荷予定日・数量・備考、「予定を登録」
- [ ] 受入処理: 一覧の「受入」で実入荷数量入力→品目在庫加算、ステータス「入荷済」
- [ ] カレンダー印刷: 当月入荷予定をA4横向き、未入荷は黄色ハイライト

### CH06 出荷管理
- [ ] 出荷待ちリスト: ステータス「完了」の受注を表示
- [ ] 受注選択で右パネルにロット情報（賞味期限古い順・FIFO、最大10件）
- [ ] 出荷日・各ロットの出荷数(c/s, p)入力、「出荷を確定する」で製品在庫減算・出荷済に更新
- [ ] アソート製品: 子製品ロットを個別に指定して出荷数入力
- [ ] バリデーション: 出荷数がロット在庫数を超える場合はエラー

### CH07 マスタ・設定
- [ ] 製品・品目・BOM・出荷先の役割説明と、Supabase/管理画面への編集案内
- [ ] BOM: 製造量(kg)あたり / 受注数(c/s)あたりの使用率。アソートは親ID(MA/FD)に共通資材、子IDに味別原材料

### CH08 FAQ・トラブル
- [ ] よくある質問の静的ページまたはアコーディオン
- [ ] 「未計画一覧に表示されない」→ 製造管理の再読み込み・既存計画の有無確認
- [ ] 「出荷待ちに出ない」→ 全製造計画が「完了」か確認
- [ ] 在庫数が合わない場合の確認ポイント（製造中での減算・入荷完了・棚卸履歴）
- [ ] 在庫アラート→ 安全在庫・入荷予定・入荷完了の案内
- [ ] 表示不具合→ キャッシュクリア・Chrome推奨・マスタ列名変更の有無

## 5. 技術メモ

- **認証**: Supabase Auth（Googleログイン推奨でマニュアルの「Googleアカウントでログイン」と整合）
- **同期**: マニュアルどおり「リアルタイム同期ではなく、ページ移動・ボタン押下で通信」を再現するか、Ver.2でリアルタイム（Realtime）を利用するか方針決定
- **ID採番**: 日付ベースの連番は Supabase の RPC または Edge Function で実装
